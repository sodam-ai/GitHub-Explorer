## M1: GitHub push (최우선)

- [ ] `main`의 로컬 전용 10커밋(보안 강화·멀티프로바이더 LLM·가져오기/내보내기·접근성 3라운드)을 origin/main에 push
- 검증: `git fetch origin && git log origin/main..HEAD --oneline` (push 전 로컬만 앞서있는지 재확인) → push 후 `git status`에 `ahead`가 사라지는지 확인
- done-when: `git status`가 `up to date with origin/main`을 표시
- 상태: pending — **사용자 명시 승인 없이는 AI가 자동 실행하지 않는 항목** (세션 내내 확립된 원칙)
- 위험/변수:
  - PUBLIC 저장소라 push 후 되돌리기 어려움(이미 클론한 사람에게 노출) → **push 직전 마지막으로 한 번 더 `git log -p` 시크릿 스캔 권장** (이번 세션 내내 시크릿 없음을 확인해왔지만 마지막 재확인은 안전장치)
  - 원격에 로컬이 모르는 커밋이 있을 가능성(사용자가 GitHub 웹에서 직접 편집했을 수 있음) → **반드시 `git fetch` 먼저, non-fast-forward면 강제 push 금지**, 이 경우 rebase/merge 여부는 재판단 필요
  - `git log`에 이미 커밋된 `889246e`(npm audit fix)는 `package-lock.json`만 변경 — push 자체는 안전 판단, 재확인 불필요

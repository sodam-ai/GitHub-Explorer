> 생성일: 2026-03-23

## 1. 제품 개요

### 한 줄 요약
GitHub 저장소/코드/이슈를 AI로 자연어 검색하고, 코드를 질문하고, 즐겨찾기로 관리하는 데스크톱 앱.

### 해결하는 문제
- GitHub 기본 검색은 키워드 기반이라 원하는 저장소를 찾기 어려움 (예: "드래그앤드롭 지원하는 React 테이블" 검색 불가)
- 코드를 이해하려면 저장소를 클론하고 직접 읽어야 함 -- 시간 소모 큼
- GitHub Stars는 분류/메모 기능이 없어서 나중에 찾기 힘듦
- 이슈/PR에서 해결책을 찾으려면 수십 개를 일일이 읽어야 함

### 핵심 가치
- **자연어 검색**: 키워드가 아닌 의미 기반으로 저장소+코드+이슈를 통합 검색
- **코드 대화**: "이 프로젝트의 인증 로직 설명해줘" 같은 질문에 AI가 코드 기반으로 답변
- **로컬 AI**: Ollama로 오프라인에서도 검색 가능, API 비용 0원
- **스마트 북마크**: GitHub Stars보다 똑똑한 폴더+메모+태그 관리

## 2. 사용자

### 주요 사용자
- **누구**: 개인 개발자 (나 혼자 사용)
- **상황**: 새 프로젝트 시작 시 라이브러리 찾을 때, 오픈소스 코드 학습할 때, 유용한 저장소 정리할 때
- **목표**: 빠르게 원하는 저장소/코드를 찾고 이해하기

### 사용자 시나리오
1. 새 프로젝트에서 드래그앤드롭 라이브러리가 필요할 때 → 앱을 열고 "React 드래그앤드롭, 스타 많고 최근 업데이트된 것" 검색
2. 찾은 라이브러리의 코드가 궁금할 때 → "이 프로젝트에서 드래그 이벤트 어떻게 처리해?" 질문
3. 마음에 드는 저장소를 "React 도구들" 컬렉션에 메모와 함께 저장
4. 나중에 컬렉션을 열어서 저장해둔 저장소들을 빠르게 확인

|------|------|----------|--------|

| 검색 결과 AI 요약 | 검색 결과 상단에 AI가 "이런 것들을 찾았어요" 1줄 요약 표시 | P1 (MVP) | 보통 |
| 북마크 & 컬렉션 | 저장소를 폴더별로 정리, 메모 추가, 검색 | P1 (MVP) | 간단 |
| 검색 히스토리 | 과거 검색 기록 저장 및 재검색 | P1 (MVP) | 간단 |
| 키보드 단축키 시스템 | Ctrl+K 검색, Ctrl+B 북마크 등 파워유저 단축키. 커맨드 팔레트 | P1 (MVP) | 보통 |

| 자연어 코드 Q&A (멀티파일) | 저장소 전체를 이해하고 질문에 답변. "프로젝트 아키텍처 설명해줘" 가능 | P2 | 복잡 |
| 저장소 건강도 분석 | 스타/커밋빈도/이슈응답속도/라이선스 종합 → 건강도 점수 표시 | P2 | 보통 |

| 코드 스니펫 뷰어 | 구문 하이라이트 + 코드 복사 | P2 | 간단 |

| 컬렉션 스마트 폴더 | 자동 분류 규칙 설정. 예: "TypeScript + 500스타 이상" → 자동 저장 | P2 | 보통 |
| 로컬 AI 지원 | Ollama로 오프라인 검색. 클라우드/로컬 AI 전환 | P3 | 보통 |
| 스마트 추천 엔진 | 내 검색/북마크 패턴을 AI가 학습하여 "이것도 마음에 들 수 있어요" 추천 | P3 | 복잡 |
| 트렌딩 대시보드 | GitHub 트렌딩 저장소/토픽을 AI가 요약. 매일 업데이트되는 개인 뉴스피드 | P3 | 보통 |
| 컬렉션 내보내기/가져오기 | JSON으로 백업 및 복원 | P3 | 간단 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름: AI 검색
```
앱 실행 -> 검색창에 자연어 입력 -> AI가 GitHub 검색 + 의미 분석
-> 저장소/코드/이슈 탭별 결과 표시 -> 마음에 드는 결과 북마크
```

### 핵심 흐름: 코드 Q&A
```
검색 결과에서 저장소 선택 -> "코드 질문하기" 클릭
-> 자연어로 질문 입력 -> AI가 코드를 읽고 답변 + 관련 코드 표시
```

### 상세 흐름
1. **앱 실행**: Tauri 데스크톱 앱 실행. 이전 검색 히스토리 표시
2. **검색**: 검색창에 자연어 입력 → GitHub API + AI 임베딩으로 통합 검색
3. **결과 탐색**: 저장소/코드/이슈 탭으로 분류된 결과 카드 확인
4. **코드 대화**: 특정 저장소 선택 → AI에게 코드 관련 질문
5. **저장**: 마음에 드는 저장소를 컬렉션에 메모와 함께 저장

## 5. 성공 기준

- [ ] 자연어 검색으로 원하는 저장소를 3개 결과 안에 찾을 수 있음
- [ ] 검색 결과 AI 요약이 1초 이내에 표시됨
- [ ] 코드 Q&A가 저장소의 실제 코드를 멀티파일로 인용하며 답변함
- [ ] Ollama 로컬 모드에서 인터넷 없이 캐시된 데이터로 검색 가능
- [ ] 컬렉션에 저장소를 추가/삭제/메모 편집이 3초 이내에 가능
- [ ] 앱 시작부터 첫 검색 결과까지 5초 이내
- [ ] 키보드만으로 검색 → 탐색 → 북마크 전체 흐름 가능
- [ ] 건강도 분석이 저장소의 활성도를 정확히 반영함
- [ ] 유사 저장소 비교표가 객관적 데이터 기반으로 생성됨
- [ ] 스마트 추천이 실제 관심사에 맞는 저장소를 제안함

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 멀티유저/팀 기능 -- 이유: 개인 전용 도구이므로
- 웹 버전 -- 이유: 데스크톱 앱으로 결정됨
- GitHub Enterprise 지원 -- 이유: 개인 사용 범위 초과
- 자동 코드 생성/수정 -- 이유: 검색+이해 도구이지 코딩 도구가 아님
- 저장소 클론/로컬 실행 -- 이유: 검색과 이해에 집중

## 7. 결정된 사항

> 모든 미결 사항이 결정되었습니다.

- [x] 클라우드 AI 제공자: **OpenAI + Anthropic 둘 다 지원** (설정에서 전환)
- [x] GitHub 인증 방식: **OAuth App** (github.com/settings/developers에서 등록)
- [x] 벡터 임베딩 저장 용량: **제한 없음** (로컬 디스크, 필요시 수동 정리)
- [x] Ollama 모델: **사용자 선택** (설치된 모델 자동 감지)
- [x] 임베딩 벡터 차원: **모델별 자동 처리** (OpenAI=1536, Ollama=모델별)
- [x] 캐시 만료 정책: **7일 자동 갱신 + 수동 새로고침**
- [x] 대화 기록 보관: **무제한** (로컬 저장, 수동 삭제 가능)
- [x] 지원 OS: **Windows 10+ / macOS 12+ / Ubuntu 22.04+**
- [x] 자동 업데이트: **GitHub Releases** (Tauri updater)
- [x] 앱 아이콘: **돋보기 + GitHub 로고** 결합 스타일
